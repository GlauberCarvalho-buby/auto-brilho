Você é um desenvolvedor sênior de Front-end especializado em React.js com TypeScript. 
Sua tarefa é criar um projeto COMPLETO, pronto para rodar, de um sistema web para um LAVA JATO brasileiro, 
seguindo os requisitos abaixo com muito cuidado.

### TECNOLOGIA E ESTRUTURA

- Use **React.js com TypeScript** obrigatoriamente.
- Estruture o projeto como um app SPA moderno, usando Vite ou Create React App (tanto faz, contanto que rode facilmente com `npm install` e `npm run dev` ou `npm start`).
- NÃO use nenhum banco de dados externo (sem MySQL, PostgreSQL, MongoDB, Firebase, etc.).
- Os dados devem ser:
  - Mockados em arquivos locais (arrays de objetos em TypeScript)
  - E/ou armazenados em **LocalStorage** para simular persistência.
- Não crie backend separado. Toda a lógica deve ser simulada no front-end.
- Todo o código, nomes de arquivos, nomes de pastas, textos da interface e comentários devem ser em **português do Brasil**.

### TEMA E CONTEXTO

O sistema será de um **lava jato brasileiro** com foco profissional, visual moderno, clean e bem organizado, 
sem cara de projeto genérico de IA. Ele deve parecer um sistema real de uso interno + site institucional.

- Identidade visual:
  - Paleta de cores inspirada em serviços automotivos profissionais de alto padrão (por exemplo: azul, branco, cinza escuro).
  - Tipografia limpa e legível.
- Utilize imagens de carros e lava jato com contexto brasileiro (pode usar URLs de imagens gratuitas de alta qualidade, como Unsplash/Pexels, 
desde que o alt text e a temática remetam ao Brasil).
- Linguagem textual profissional, porém acessível. Nada de inglês no texto da interface.

### PAPÉIS E AUTENTICAÇÃO

Implemente um sistema de login SIMULADO com **três tipos de usuário**:

1. **Cliente**
2. **Funcionário**
3. **Administrador**

- Os usuários podem ser definidos em um arquivo `usuariosMock.ts` ou semelhante, com estrutura de tipo `Usuario` (id, nome, email, senha, papel).
- A autenticação deve ser SIMULADA:
  - Verificar email e senha contra a lista mock.
  - Ao logar, salvar as informações básicas do usuário logado em LocalStorage (por exemplo: token fake + papel + nome).
- Implementar:
  - **Tela de Login** para todos os tipos de usuário.
  - **Tela de Cadastro de Cliente** (apenas clientes podem se cadastrar; funcionários e admins são mockados).
  - Lógica de logout limpando LocalStorage.
- Implementar proteção de rotas:
  - Algumas rotas só podem ser acessadas por usuários logados.
  - Algumas rotas só podem ser acessadas por ADMIN.
  - Outras só por FUNCIONÁRIO.
  - Cliente logado vê seu próprio painel.

### HOOKS (PELO MENOS 4 DIFERENTES)

Use **pelo menos 4 hooks diferentes**, incluindo:

- `useState` – para estados locais (inputs de formulário, status, etc.).
- `useEffect` – para carregar dados do LocalStorage, salvar alterações, etc.
- `useContext` – para contexto de autenticação (AuthContext) e, se quiser, contexto de agendamentos.
- `useRef` – por exemplo, para focar automaticamente no primeiro campo do login ou para rolar para uma seção específica.
- `useMemo` – opcional, para memorizar listas filtradas (ex: agendamentos filtrados por status ou data).
- Crie **pelo menos um custom hook**, por exemplo:
  - `useAuth` para lidar com login, logout, usuário atual.
  - Ou `useAgendamentos` para abstrair lógica de agendamentos.
